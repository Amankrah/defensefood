Tier 2 — Corridor Construction & Entity Matching
Corridor Definition Engine
Each RASFF notification r maps to corridors (c, i, j) where: c = commodity (HS-code matched), j = Origin(r), i ∈ Affected(r) = {notifying_country ∪ for_attention ∪ for_followUp}.
Each corridor is then enriched with its Comtrade trade flow and FAOSTAT production/consumption data.
Entity: Commodity × Origin × Destination
Tier 3 — Mathematical Model Layer
Dependency Models
DS′ = P + M − X
IDR = M(c,i,•,t) / DS′
OCS = M(c,i,j,t) / M(c,i,•,t)
BDI = IDR × OCS
HHI = Σⱼ [OCS]²
SCI = IDR × OCS × (1+HHI)
§2 Dependency
Consumption Models
PCC = D(c,i,t) / Pop(i,t)
CRS = 1 − (Rank−1)/(|C|−1)
CV_D = σ[PCC] / μ[PCC]
DIS = 1 − min(CV_D, 1)
§3 Consumption
Hazard Signal Models
S(r) = W_class × W_risk
HIS = Σᵣ S(r) · α^(t−tᵣ)
HDI = −Σₕ pₕ ln(pₕ)
DGI = trade_share − notif_share
§4 Hazards
Trade Flow Models
UV = V / M (unit value)
Z_UV = (UV − μ) / σ
Z_M = volume anomaly z-score
MTD = mirror trade discrepancy
ΔHHI, ΔOCS shifts

Tier 3b — Origin–Attention Country Network
Exposure Network Graph G = (V, E)
Directed weighted graph: edges (j → i) carry w_trade, w_hazard, w_dep per commodity.
ORPS(j,c,t) = Σᵢ BDI · HIS · PCC — origin risk propagation across EU
ACEP(i,t) = Σ_c Σ_j BDI · HIS · CRS — total inbound exposure per country
P̂(hazard|c,i,j) = R(c,i,j,T) / [M(c,i,j,T) / m̄(c)] — empirical detection rate

Tier 4 — Normalisation & Composite Scoring
Corridor Vulnerability Score (CVS)
Normalise each metric → percentile rank (log-percentile for HIS)
CVS = [SCI_norm × CRS_norm] × (1 + wₕ·HIS_norm + wₚ·PAS_norm + w_sc·SCCS_norm)
BASE (structural dependency × demand) × (1 + AMPLIFIER (hazard + price + supply chain signals))
§7 · Weights: AHP / Equal / Empirical calibration
CVS Classification Thresholds
● Critical 80–100 ● High 60–79 ● Elevated 40–59 ● Moderate 20–39 ● Low 0–19
Tier 5 — Intelligence Engines
Anticipation Engine
Before incident

Structural scanning: High SCI + high CRS but no RASFF history → under-detected corridor

Leading indicators: Trade surges (Z_M > 2), price drops (Z_UV < −2), dependency shifts (ΔOCS), rising low-severity notifications
Proactive
Detection Engine
During incident

RASFF processing: New notification → immediate corridor enrichment with all layer context

Cross-layer correlation: Trade surge + price drop + rising notifications = convergent fraud signal
Real-time
Countering Engine

Post-incident: Retrospective validation → weight adjustment → analogous corridor identification
